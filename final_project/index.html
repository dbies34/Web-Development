CPSC 314 Final Project Presentation

Social Site

Drew Bies

I planned for Social Site to be a social media website similar to Facebook. The website allows for users to create an account and login to it. They can then post content and look at their friend's content.

Functional Requirements:

  1. Users must be able to register a username and password

  2. Users must be able to log in with a registered a username and password

  3. Users must be able to look at other users’ content. When a user looks at another user’s profile, they will be able to see the content they have posted.

  4. Users’ profile information is stored correctly. When a user logs on, all their profile information is the same as they last saw it.

  5. Users must be able to post a form request to the mongo database when they make a new account. Users are able to store their username and password into the database.

  6. Users must be able to edit their profile information. They must be able to go to their profile and use the edit button to make changes to their profile and have it save.

  7. Users must be able to delete their account. In the profile page, there will be an option to delete your profile and all your information will be wiped from the database.

  8. Users must be able to add a friend. They can use the add friend button on another user’s account and add that user to their friend list.

  9. Users must be able to send a message to another user. They can use the message button located on the other user’s page to send a direct message to them.

  10. Users must be able to post content. When a user post content that information will be saved to the database and will saved for them.
